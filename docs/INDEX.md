# Documentation Index

This index keeps project documentation discoverable for operators and for the Ops Console ingestion contract.

## Root documents

- `README.md` — local setup, stack, and verification entrypoint.
- `AGENTS.md` — repository operating instructions. Keep this at repo root so agents can load it automatically.

## Control documents

- `docs/control/TEST_PLAN.md` — verification checklist and latest test evidence.
- `docs/control/DECISIONS.md` — architecture and operational decisions.
- `docs/control/PROGRESS.md` — implementation progress and validation notes.
- `docs/control/SESSION_HANDOFF.md` — current handoff state and next work.
- `docs/control/SPEC.md` — current spec snapshot.

## Frontend documents
